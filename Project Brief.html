Project Brief

Pure Dental People Portal

A secure, role-based HR & onboarding platform for Pure Dental, Four Ever Smile & Pure Dental Lab.
Status · Active prototype
Updated · Jun 2026
Stack · React + Azure

The People Portal is a single, login-gated home for the entire employee lifecycle — hiring, onboarding, scheduling, time, learning, reviews and offboarding — that adapts what each person sees to their role. It runs on Google Workspace sign-in, a server-scoped roster, and a modular React front end. This brief captures everything currently built, what is live versus simulated, and what remains.

01

What it is

Secure by role

Google Workspace SSO, domain-locked. Seven access levels drive every nav item, view and action.

One roster, scoped

The server returns only the people a user may see — all, their reporting tree, or just themselves.

Modular & phased

~20 feature modules, each behind a rollout flag so the platform turns on capability by capability.

02

Access model

Sign-in is restricted to puredental.com, foureversmile.com and puredentallab.com. After Google verifies identity, access is derived from the person's title, department and explicit permission grants — then the roster is scoped to match.

Administrator
Full control. Manages users & permissions, feature rollout, and can delete records.
HR & Payroll
Sees everyone. Hiring, offboarding approval, reviews, payroll/time, compliance.
Leadership
Sees everyone. Org-wide visibility and reports, without user administration.
Accounting
Payroll & reports. Time, overtime and cost analytics.
Manager
Sees their reporting tree. Scheduling, prehire requests, team reviews & offboarding.
Supervisor
Sees their team. Scheduling and team visibility without hire/terminate rights.
Employee
Sees themselves. Onboarding, schedule, time clock, learning, scrubs, reviews.
03

Capabilities

Every module below is built and navigable in the app today. Most run on live in-app logic; integration-dependent features are flagged for later phases.

Hiring & onboarding

Applicant Tracking (ATS)

Hiring pipeline board — Applied → Screening → Interview → Offer → Hired — with applicant detail, manual add, résumé import, and a one-click hand-off straight into onboarding. PII encrypted at rest.

Guided onboarding

Step-by-step new-hire flow: welcome, details intake, e-signature paperwork, provider credentials, policies, accounts & access, training, team, schedule and benefits. Task templates adapt to role.

Onboarding automations & agent

Add a hire and a multi-channel agent takes over — voice-call transcript, Google Chat collaboration feed, routing directory — verifying credentials and provisioning role-based accounts as a live pipeline.

Provider credentials

Verify-first capture of NPI, license and DEA — type an ID, look it up, auto-fill the rest.

Daily operations

Scheduling

Deputy-style builder: draft → publish, copy last week or pay period, role-based coverage targets, an open-shifts lane (create / assign / claim) and drag-to-swap. Employees get a read-only personal schedule.

Time clock

Clock in/out by location with a live timer, breaks, timesheets, manager/HR approval and export to Paychex.

Directory & records

Contact and role for everyone; full employee records — overview, relations, documents, offboarding — gated by access.

Offices

View all offices with live headcount per location, and add new ones.

Growth, people & perks

Performance reviews

HR/leadership define question sets; employees self-review and managers review; results gated by role.

Learning Library

Training catalog with required/recommended items, due dates and reminders, per-team or individual assignment, plus transcribed clerical & clinical orientation timelines.

Scrubs ordering

Staff storefront with an eligibility engine (90-day gate, annual April reset), cart, and an ops pipeline — approve → batch to CID Resources → receive → deliver — with invoices and analytics.

Offboarding

Termination requests with scoped visibility, manager initiation and HR approval.

Celebrations

Birthday & work-anniversary moments at sign-in, with Sunshine Club helpers.

Reports

Drag-and-drop report builder, Time & Overtime analytics, and scheduled email reports.

Assistance & experience

Ask Riley

Employee-facing AI assistant for portal and policy questions, with optional voice narration.

Ask HR

Management-facing outsourced-HR advisor (CEDR-style): advisor chat, guides & templates, webinars and jurisdiction-specific compliance alerts.

Help & guided tour

AI-presenter walkthrough plus a dismissible interactive tour available on every page.

Roadmap & feedback

Anyone can submit and view feature requests; admins set status and publish a visual roadmap.

Personalization

Per-employee appearance from a hidden menu — accent, three tint depths, light/dark, density, corners, text size, and five themed font pairings (each weight-tuned) — plus preferences like language and Riley's voice.

Localization

Runtime translation that swaps interface text per language; new languages can be added on the fly.

04

How it's built

  • Front endReact 18 with in-browser Babel, split across ~40 modular JSX files — no build step. Theming via CSS design tokens (OKLCH color, tweakable accent/density/corners).
  • Hosting / APIAzure Static Web Apps + Functions. Read endpoint for the roster, AI transport, and signed write endpoints for scheduling, coverage, settings, time-off and org config.
  • IdentityGoogle Workspace SSO. Token verified server-side (signature, audience, expiry); email domain-locked to the three company domains.
  • DatabaseAzure Cosmos DB (portal) — collections for roster, app state, schedules, coverage requirements and per-employee settings. Writes go through one shared signed-upsert helper with etag concurrency.
  • Sign-in flowBrowser signs in → calls /api/roster with the token → server verifies, domain-locks, reads Cosmos, scopes the roster to the caller → app renders from it.
  • SecuritySensitive in-browser data (applicant contact details, résumés, notes) stored as AES-256-GCM ciphertext. Secrets held as Functions environment variables, never in code.
05

Live vs. simulated

An honest read of what's real today. The interface and logic are fully built; several external integrations are stubbed so the experience is complete end-to-end before the APIs are wired up.

Paperwork e-sign & Paychex syncSimulatedSigning flow and payroll export are mocked.
Integration flagsSimulatedPaychex, Workspace/Denticon/NexHealth/DoseSpot provisioning, Google Drive, résumé parsing — no live API yet.

The write path is live. A shared signed-upsert helper (api/_shared/cosmos.js) backs authenticated endpoints for settings, schedules, coverage, time-off and org config — each enforcing token, domain lock and role before writing. What remains is migrating the last localStorage-only domains (photos, forms, punches, reviews, orders, applicants, automations) onto the same pattern.
