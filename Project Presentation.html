01 · What it is

A single portal that adapts to who's looking.

Sign in once with a company Google account and the portal shows exactly what your role needs — nothing more, nothing less.

Secure by role

Google Workspace sign-in, domain-locked. Seven access levels drive every screen, menu and action.

One roster, scoped

The server returns only the people you may see — everyone, your reporting tree, or just you.

Modular & phased

Around twenty modules, each behind a rollout flag, so capability turns on one piece at a time.

04 · How it's built

A modular front end on Azure, scoped at the door.

Front end
React 18 · modular JSX
~40 files, no build step. OKLCH design tokens, tweakable theme.
Identity
Google Workspace SSO
Token verified server-side. Domain-locked to the three companies.
API · Azure Functions
Roster · AI · Schedule · Settings
Verifies, domain-locks, derives access, scopes the roster to the caller.
Database
Azure Cosmos DB
Collections for roster, app state, schedules & coverage.
Security
AES-256-GCM at rest
Sensitive in-browser PII encrypted. Secrets in Functions env, never in code.
Sign in → /api/roster with the token → server verifies, domain-locks and reads Cosmos → returns only the people you're allowed to see → the app renders from it.
